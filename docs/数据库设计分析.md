# WebQuest 数据库设计分析

## 1. 分析目标

本文档系统梳理 `WebQuest` 项目所有需要数据持久化的模块，明确 MVP 阶段必须建立的数据表，以及预留的扩展空间，避免后续反复返工。

---

## 2. 数据需求总览（按业务模块）

### 2.1 用户系统

| 数据需求 | 来源 | 当前状态 |
|---------|------|---------|
| 用户身份（ID、用户名、密码） | auth.service / users.service | mock-db |
| 用户成长数据（等级、经验值） | auth.service / submissions.service | 存在 users 中 |
| 个人资料（头像、昵称、注册时间） | MVP 简化，暂缓 | 预留 |

**核心用途**：注册、登录、鉴权、关卡页展示用户信息。

### 2.2 地图区域系统

| 数据需求 | 来源 | 当前状态 |
|---------|------|---------|
| 区域定义（ID、名称、描述） | zones.service | mock-db |
| 区域解锁条件（需要等级） | mock-db | mock-db |
| 区域排序与展示顺序 | 未定义 | 需要补充 |

**核心用途**：世界地图页展示区域列表和锁定状态。

### 2.3 关卡系统

| 数据需求 | 来源 | 当前状态 |
|---------|------|---------|
| 关卡定义（ID、标题、描述、所属区域） | levels.service | mock-db |
| 关卡难度、奖励 XP | mock-db | mock-db |
| 初始代码模板（html/css/javascript） | levels.service | mock-db |
| 通关校验规则（requiredKeywords） | submissions.service | mock-db |
| 关卡剧情/背景故事 | spec 要求 | 当前缺失 |
| 关卡顺序与依赖关系 | 未定义 | 需要补充 |

**核心用途**：关卡选择页展示、提交判题。

### 2.4 提交记录系统

| 数据需求 | 来源 | 当前状态 |
|---------|------|---------|
| 提交 ID、用户 ID、关卡 ID | submissions.service | mock-db |
| 用户提交的代码 | submissions.service | mock-db |
| 判题结果、得分 | submissions.service | mock-db |
| 提交时间 | submissions.service | mock-db |

**核心用途**：记录用户每次闯关结果、判题回溯、学习历史。

### 2.5 用户进度系统

| 数据需求 | 来源 | 当前状态 |
|---------|------|---------|
| 用户已通关关卡列表 | progress.service / submissions.service | mock-db（数组） |
| 用户已解锁区域列表 | progress.service / auth.service | mock-db（数组） |
| 用户当前所在关卡 | progress.service | mock-db |
| 用户等级与 XP | submissions.service | 存在 users 表 |

**核心用途**：地图解锁、关卡状态、通关反馈。

### 2.6 XP 日志系统

| 数据需求 | 来源 | 当前状态 |
|---------|------|---------|
| XP 变动记录（谁、因为什么、变动了多少） | spec 约束要求 | 当前缺失 |
| 变动时间 | spec 约束要求 | 当前缺失 |

**核心用途**：成长记录回溯、防止奖励被篡改、后续排行榜基础。

### 2.7 技能树系统（MVP 展示型）

| 数据需求 | 来源 | 当前状态 |
|---------|------|---------|
| 技能节点定义 | spec 9.6 | 当前缺失 |
| 技能节点与关卡的关联 | spec 9.6 | 当前缺失 |
| 用户技能解锁记录 | spec 9.6 | 当前缺失 |

**核心用途**：技能树页面展示成长路径。

## 4. 表结构详细设计

### 4.1 users（用户）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 哈希密码 |
| nickname | VARCHAR(50) | nullable | 昵称（预留） |
| avatar_url | VARCHAR(255) | nullable | 头像（预留） |
| level | INT | NOT NULL DEFAULT 1 | 当前等级 |
| xp | INT | NOT NULL DEFAULT 0 | 当前总经验值 |
| status | VARCHAR(20) | DEFAULT 'active' | 账号状态 |
| created_at | TIMESTAMP | NOT NULL | 注册时间 |
| updated_at | TIMESTAMP | NOT NULL | 更新时间 |

**当前业务关联**：
- auth.service：注册写入、登录读取
- users.service：读取用户资料
- submissions.service：更新等级和 XP
- progress.service：读取用户等级

### 4.2 zones（地图区域）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| name | VARCHAR(50) | NOT NULL | 区域名称（新手村、CSS 森林等） |
| slug | VARCHAR(50) | UNIQUE, NOT NULL | 英文标识 |
| description | TEXT | nullable | 区域描述 |
| icon | VARCHAR(100) | nullable | 区域图标（预留） |
| sort_order | INT | NOT NULL DEFAULT 0 | 展示排序 |
| required_level | INT | NOT NULL DEFAULT 1 | 解锁需要等级 |
| status | VARCHAR(20) | DEFAULT 'active' | 状态 |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

**当前业务关联**：
- zones.service：全量读取
- mock-db 中已有 3 个区域，预留 4 个区域

### 4.3 levels（关卡）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| zone_id | VARCHAR(36) | FK → zones.id | 所属区域 |
| title | VARCHAR(100) | NOT NULL | 关卡标题 |
| description | TEXT | nullable | 关卡描述/通关目标 |
| story | TEXT | nullable | 剧情背景（当前 mock-db 缺失） |
| difficulty | ENUM('easy','medium','hard') | NOT NULL | 难度 |
| sort_order | INT | NOT NULL | 区域内排序 |
| reward_xp | INT | NOT NULL DEFAULT 10 | 通关奖励 XP |
| starter_code_html | TEXT | nullable | 初始 HTML 代码 |
| starter_code_css | TEXT | nullable | 初始 CSS 代码 |
| starter_code_js | TEXT | nullable | 初始 JS 代码 |
| required_keywords | JSON | nullable | 通关需要的关键词列表 |
| status | VARCHAR(20) | DEFAULT 'active' | 状态 |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

**当前业务关联**：
- levels.service：关卡列表（支持按区域筛选）、关卡详情
- submissions.service：读取关卡定义用于判题

**为什么 required_keywords 用 JSON**：
- 关卡之间关键词数量不固定
- 后续升级判题规则时不影响已有数据
- 避免为关键词建一张单独的表，增加复杂度

### 4.4 submissions（提交记录）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| user_id | VARCHAR(36) | FK → users.id | 提交用户 |
| level_id | VARCHAR(36) | FK → levels.id | 提交关卡 |
| code_html | TEXT | nullable | 用户提交的 HTML |
| code_css | TEXT | nullable | 用户提交的 CSS |
| code_js | TEXT | nullable | 用户提交的 JS |
| passed | BOOLEAN | NOT NULL | 是否通关 |
| score | INT | NOT NULL DEFAULT 0 | 得分（0-100） |
| earned_xp | INT | NOT NULL DEFAULT 0 | 获得 XP |
| submitted_at | TIMESTAMP | NOT NULL | 提交时间 |

**当前业务关联**：
- submissions.service：每次提交产生一条记录

### 4.5 user_level_progress（用户关卡进度）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| user_id | VARCHAR(36) | FK → users.id, UNIQUE | 用户 |
| current_level_id | VARCHAR(36) | FK → levels.id | 当前所在关卡 |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

**关联表：user_completed_levels**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| user_id | VARCHAR(36) | FK → users.id | 用户 |
| level_id | VARCHAR(36) | FK → levels.id | 已通关关卡 |
| completed_at | TIMESTAMP | NOT NULL | 通关时间 |
| UNIQUE(user_id, level_id) | | 防止重复记录 |

**关联表：user_unlocked_zones**

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| user_id | VARCHAR(36) | FK → users.id | 用户 |
| zone_id | VARCHAR(36) | FK → zones.id | 已解锁区域 |
| unlocked_at | TIMESTAMP | NOT NULL | 解锁时间 |
| UNIQUE(user_id, zone_id) | | 防止重复记录 |

**设计原因**：
- 将 `completedLevelIds: string[]` 和 `unlockedZoneIds: string[]` 拆为关联表
- 符合关系型数据库第三范式
- 支持后续查询“谁通关了哪些关”、“每个关卡的通过人数”等分析

### 4.6 xp_logs（XP 日志）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| user_id | VARCHAR(36) | FK → users.id | 用户 |
| source | VARCHAR(50) | NOT NULL | XP 来源（'submission', 'bonus'） |
| source_id | VARCHAR(36) | nullable | 来源 ID（如 submission_id） |
| amount | INT | NOT NULL | 变动数量（正数=增加） |
| balance_after | INT | NOT NULL | 变动后余额 |
| created_at | TIMESTAMP | NOT NULL | 变动时间 |

### 4.7 skills（技能树节点——MVP 展示型）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| name | VARCHAR(100) | NOT NULL | 技能名称 |
| description | TEXT | nullable | 技能说明 |
| category | VARCHAR(50) | NOT NULL | 分类（html/css/javascript） |
| parent_id | VARCHAR(36) | FK → skills.id, nullable | 父节点 |
| level_id | VARCHAR(36) | FK → levels.id, nullable | 关联关卡 |
| sort_order | INT | DEFAULT 0 | 排序 |
| created_at | TIMESTAMP | NOT NULL | |
| updated_at | TIMESTAMP | NOT NULL | |

### 4.8 user_skills（用户技能——MVP 展示型）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(36) | PK | UUID |
| user_id | VARCHAR(36) | FK → users.id | 用户 |
| skill_id | VARCHAR(36) | FK → skills.id | 技能节点 |
| unlocked_at | TIMESTAMP | NOT NULL | 解锁时间 |
| UNIQUE(user_id, skill_id) | | 防止重复 |

## 6. 当前 mock-db 到数据库的映射关系

| mock-db 数据 | 对应的数据库表 | 迁移方式 |
|-------------|--------------|---------|
| users | users | 直接映射，password 改为 password_hash |
| zones | zones | 直接映射 |
| levels | levels | starterCode 拆为 3 个字段 |
| submissions | submissions | code 拆为 code_html/css/js |
| progressRecords | user_level_progress + user_completed_levels + user_unlocked_zones | 拆为 3 张表 |
| xp_logs（缺失） | xp_logs | 需新建 |
| skills（缺失） | skills | 需新建 |

## 7. 当前 mock-db 中缺失但 MVP 需要的数据

| 缺失内容 | 影响 | 建议 |
|---------|------|------|
| 关卡剧情（story） | 关卡页没有剧情展示 | levels 表加 story 字段 |
| 区域图标/颜色 | 地图页视觉区分 | zones 表加 icon 或 color 字段 |
| 关卡排序 | 关卡列表顺序不确定 | levels 表加 sort_order |
| 区域排序 | 区域顺序不确定 | zones 表加 sort_order |
| XP 日志 | 违反 spec 约束 | 新建 xp_logs 表 |
| 技能树数据 | 技能树页无数据 | 新建 skills + user_skills 表 |
| 首关关联 | 新用户进入后没有引导 | user_level_progress 初始化时关联首关 |

## 9. 技术选型建议

| 层面 | 选择 | 理由 |
|------|------|------|
| 数据库 | **MySQL** | 前文指定，社区成熟，Prisma 支持完善 |
| ORM | **Prisma** | 类型安全、迁移管理、Seed 支持 |
| 连接方式 | 单连接池 | MVP 阶段无需读写分离 |
