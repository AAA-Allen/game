# WebQuest 完整关卡体系设计

## 设计理念

### 核心原则
- **知识全分解**：每个知识点拆到最小粒度，一个子知识点 = 至少一个关卡
- **层层递进**：每一关都依赖前一关，前几关是后几关的认知基础
- **环环相扣**：早期关卡埋下的知识点，在后面关卡中反复使用和深化
- **认知闭环**：每 3-5 关形成一个学习循环（概念→模仿→练习→应用→综合）

### 关卡类型定义
| 类型 | 说明 | 提示程度 | 目标 |
|------|------|----------|------|
| 教学关 | 展示概念，用户只需补充 1-2 行 | 详细 | 建立认知 |
| 模仿关 | 给目标效果，用户照着还原 | 中等 | 理解用法 |
| 练习关 | 给需求描述，用户独立完成 | 少量 | 掌握应用 |
| 修复关 | 给错误代码，用户找 Bug | 提示 Bug 位置 | 加深理解 |
| 填空关 | 在指定位置补充关键代码 | 仅提示上下文 | 聚焦核心 |
| 应用关 | 多个知识点混合使用 | 无详细提示 | 综合能力 |
| Boss 关 | 章节综合挑战 | 仅剧情需求 | 验收掌握 |

### 关详细设计

#### 1.1 页面骨架（3关）

**第1关：认识 HTML 结构**
- 类型：教学关
- 前置：无
- 目标：写出 `<!DOCTYPE html>` / `<html>` / `<head>` / `<body>` 的骨架
- 任务：在编辑器中补全缺少的标签，让页面结构完整
- 提示：给出大部分代码，只去掉 `<head>` 和 `<body>` 让用户补充
- 通关判定：验证存在 `<head>` / `<body>` / `<!DOCTYPE html>`
- 关键代码片段：`"<!DOCTYPE", "<html", "<head>", "<body>"`
- XP：10

**第2关：给页面加上标题**
- 类型：教学关
- 前置：第1关
- 目标：在 `<head>` 里加 `<title>`，在 `<body>` 里加 `<h1>`
- 任务：在骨架完成后，添加页面标题和主标题
- 通关判定：存在 `<title>` 标签且内容非空，存在 `<h1>` 标签
- 关键代码片段：`"<title>`, `"<h1>"`
- XP：10

**第3关：修复一个不完整的页面**
- 类型：修复关
- 前置：第1-2关
- 目标：修复缺少闭合标签和标题的 HTML
- 任务：给一段缺少 `</body>` `</html>` 和 `<title>` 的代码修复
- 通关判定：文档结构完整，标签正确闭合，存在 `<title>`
- 关键代码片段：`"</html>"`, `"</body>"`, `"<title>", "</title>"`, `"<!DOCTYPE"`
- XP：15

## 解锁机制

### 区域解锁
| 区域 | 前置要求 | 解锁条件 |
|------|----------|----------|
| 新手村（HTML） | 无 | 注册后自动解锁 |
| CSS 森林（CSS 基础） | HTML 通关 ≥ 60% | 完成第 1-28 关 ≥ 17 关 |
| 布局峡谷（CSS 布局） | CSS 基础通关 ≥ 60% | 完成第 29-56 关 ≥ 17 关 |
| JS 风暴（JS 基础） | CSS 基础通关 ≥ 50% | 完成第 29-56 关 ≥ 14 关 |
| 交互遗迹（DOM） | JS 基础通关 ≥ 60% | 完成第 79-104 关 ≥ 16 关 |
| 全栈试炼（综合） | 全部前置完成 | 全部前置区域解锁 |

### 关卡解锁
- 基础规则：前置关卡完成后自动解锁下一关
- 跳跃解锁：连续通关 3 关后，额外解锁 1 关
- Boss 关：需要该章全部前置关卡完成
