# WebQuest 需求文档

## 1. 文档信息

- 项目名称：`WebQuest`
- 文档类型：产品需求文档（PRD）
- 当前版本：`V1.0`
- 项目阶段：设计阶段 / 待开发

## 2. 项目背景

传统 Web 开发学习方式通常以教程阅读、视频学习和零散练习为主，存在以下问题：

- 学习路径枯燥，缺乏持续动力
- 知识点割裂，难以形成完整能力体系
- 学习反馈滞后，缺乏即时成就感
- 项目实战门槛高，新手难以坚持

为解决上述问题，`WebQuest` 通过 RPG 游戏化设计，将 Web 开发知识拆解为地图区域、任务关卡、技能树与成长系统，使用户在“冒险闯关”中逐步掌握前端与后端开发能力。

## 3. 项目目标

### 3.1 产品目标

- 构建一个以游戏闯关为核心的 Web 开发学习平台
- 帮助初学者以更低门槛完成 HTML、CSS、JavaScript 的系统学习
- 通过即时反馈、等级成长和地图解锁提升学习粘性

### 3.2 MVP 目标

- 完成基础世界地图 UI
- 提供 10 个可体验的前端开发关卡
- 支持 HTML/CSS/JavaScript 在线运行与预览
- 支持用户登录、进度保存、XP 增长与等级计算

## 4. 目标用户

### 4.1 用户画像

- Web 开发初学者
- 前端入门学生
- 想以项目化、互动化方式学习编程的用户
- 需要演示型产品或教学项目的学生团队

### 4.2 用户痛点

- 不知道从哪里开始学 Web
- 做练习时没有明确目标和反馈
- 学完语法后不会实际应用
- 很难长期坚持

## 5. 产品定位

`WebQuest` 是一个将 Web 开发学习内容与 RPG 游戏机制融合的互动学习平台。平台核心不是单纯提供课程，而是通过“地图探索 + 任务闯关 + 在线编码 + 成长奖励”的方式，让用户在持续互动中完成能力提升。

## 6. 核心价值

- 将抽象的技术学习路径可视化
- 将练习任务转化为具备目标和奖励的游戏关卡
- 让学习过程具备即时反馈和成长感
- 提升用户学习动力与完成率

## 7. 产品范围

### 7.1 MVP 范围

本阶段必须实现：

- 世界地图页面基础展示
- 新手村、CSS 森林、JS 风暴三个核心区域
- 10 个前端开发学习关卡
- 关卡详情页
- 在线代码编辑器
- HTML/CSS/JS 实时运行与预览
- 用户提交记录
- XP 与等级系统
- 关卡解锁逻辑

### 7.2 非 MVP 范围

本阶段不实现：

- AI 代码纠错与智能教练
- 排行榜系统
- 多角色或职业系统
- 真实后端隔离运行环境
- 多人联机或社交功能
- 高级粒子动画和复杂特效

## 8. 产品功能结构

### 8.1 一级模块

- 用户系统
- 世界地图系统
- 关卡战斗系统
- 在线代码运行系统
- XP 等级系统
- 技能树系统

### 8.2 模块关系说明

- 用户登录后进入世界地图
- 在地图中选择已解锁区域和关卡
- 进入关卡页完成任务
- 通过在线编辑器编写代码并运行预览
- 提交后获得判定结果
- 通关后增加 XP、提升等级并解锁新关卡

## 9. 功能需求明细

### 9.1 用户系统

#### 功能说明

用于用户身份识别、学习进度保存和基础资料展示。

#### 功能点

- 用户注册
- 用户登录
- 用户退出登录
- 获取个人信息
- 保存当前等级、XP、已通关关卡与技能解锁状态

#### MVP 约束

- 可采用简化登录方式
- 暂不支持第三方登录

### 9.2 世界地图系统

#### 功能说明

以游戏地图形式承载 Web 学习路径，按区域呈现技术模块。

#### 功能点

- 地图主页面展示
- 不同区域的视觉区分
- 区域锁定与解锁状态展示
- 当前可挑战关卡高亮
- 点击进入关卡

#### 地图区域规划

- 新手村：HTML 基础
- CSS 森林：样式与布局
- JS 风暴：JavaScript 核心
- 前端框架城：预留区域
- 后端王国：预留区域
- 数据库遗迹：预留区域
- 全栈 Boss 战：预留区域

### 9.3 关卡战斗系统

#### 功能说明

每个关卡对应一个 Web 开发任务，用户需要根据任务目标完成代码编写并提交。

#### 功能点

- 展示任务背景与目标
- 展示任务提示
- 提供初始代码模板
- 支持运行与提交
- 返回通关结果、评分与奖励

#### 示例关卡

- 制作登录页面
- 实现点击计数器
- 完成导航栏布局
- 实现按钮交互效果
- 实现简单 DOM 操作

### 9.4 在线代码运行系统

#### 功能说明

支持用户在浏览器中直接编写并运行 HTML、CSS、JavaScript 代码。

#### 功能点

- 代码编辑器
- 多代码区域或合并代码区域
- 运行按钮
- 预览窗口
- 错误提示
- 重置代码

#### MVP 实现方式

- 通过前端沙箱或 `iframe` 预览 HTML/CSS/JS 运行结果
- 暂不支持 Java、Spring Boot、Node 服务端真实运行

### 9.5 XP 与等级系统

#### 功能说明

通过经验值奖励强化用户成就感，并驱动关卡与区域逐步解锁。

#### 功能点

- 通关奖励 XP
- 连续通关奖励
- 等级计算
- 等级展示
- 解锁下一关卡或区域

#### 等级示例

- `Lv1-Lv10`：基础学习阶段
- `Lv11-Lv30`：进阶探索阶段
- `Lv31-Lv60`：实战提升阶段
- `Lv61-Lv100`：高阶挑战阶段

### 9.6 技能树系统

#### 功能说明

将 Web 开发知识以成长路径形式展示，增强用户对能力提升的感知。

#### MVP 建议

- 首版以展示型技能树为主
- 技能节点可与关卡通关状态关联
- 暂不实现复杂技能分支属性系统

## 10. 关键业务流程

### 10.1 核心学习流程

1. 用户登录平台
2. 进入世界地图页
3. 选择已解锁关卡
4. 查看任务剧情和目标
5. 编写代码并运行预览
6. 提交任务
7. 系统判定结果
8. 用户获得 XP、等级变化与解锁反馈

### 10.2 提交流程

1. 用户点击提交
2. 系统校验当前代码结果
3. 若通过，记录提交结果并发放 XP
4. 若未通过，返回失败原因或提示

## 11. 页面需求

### 11.1 世界地图页

#### 页面目标

作为平台主入口，承载区域导航、进度查看与关卡入口。

#### 页面内容

- 地图背景
- 区域模块
- 解锁状态
- 用户等级与 XP 信息
- 当前任务提示

### 11.2 关卡页面

#### 页面目标

承载任务阅读、代码编辑、运行预览与任务提交。

#### 页面布局建议

```text
┌──────────────┬──────────────┬──────────────┐
│ 任务剧情区   │ 代码编辑器   │ 实时预览区   │
└──────────────┴──────────────┴──────────────┘
```

#### 页面内容

- 关卡标题
- 背景剧情
- 通关目标
- 代码编辑器
- 运行按钮
- 提交按钮
- 实时预览
- 错误反馈 / 成功反馈

### 11.3 通关结算页

#### 页面目标

向用户展示本次挑战结果，并强化奖励反馈。

#### 页面内容

- 通关提示
- 星级评分
- XP 奖励
- 等级变化
- 解锁内容
- 下一关入口

## 12. 非功能需求

### 12.1 性能需求

- 页面首屏加载合理
- 代码运行与预览反馈及时
- 主要交互响应流畅

### 12.2 可用性需求

- 新手可以在较短时间理解页面结构
- 任务目标说明必须清晰
- 提交失败时必须有明确反馈

### 12.3 安全需求

- 用户数据必须与账号关联
- 提交与奖励逻辑不能完全由前端决定
- 基础鉴权与接口权限控制需具备

### 12.4 可扩展性需求

- 保留前端框架城、后端王国、数据库遗迹扩展空间
- 系统结构需支持未来新增关卡、地图区域和成长机制

## 13. 数据需求

### 13.1 核心数据对象

- 用户
- 地图区域
- 关卡
- 提交记录
- XP 日志
- 技能树节点

### 13.2 建议数据字段

#### 用户

- 用户 ID
- 用户名
- 密码
- 当前等级
- 当前 XP

#### 关卡

- 关卡 ID
- 关卡名称
- 所属区域
- 关卡描述
- 初始代码
- 通关规则
- 奖励 XP

#### 提交记录

- 提交 ID
- 用户 ID
- 关卡 ID
- 提交代码
- 运行结果
- 是否通过
- 提交时间

## 14. MVP 关卡规划建议

推荐首批 10 关如下：

1. 创建基础 HTML 页面
2. 制作个人介绍卡片
3. 实现登录页面布局
4. 制作导航栏
5. 完成 Flex 布局页面
6. 实现按钮点击提示
7. 制作点击计数器
8. 实现表单输入读取
9. 完成 DOM 内容修改
10. 综合练习：交互式任务卡片

## 15. 验收标准

### 15.1 产品验收

- 用户可成功注册并登录
- 世界地图可正常显示区域和解锁状态
- 至少 10 个关卡可正常进入
- 用户可在编辑器中编写 HTML/CSS/JS 代码
- 运行结果可在预览区展示
- 提交后系统可正确判定通关结果
- XP 与等级可正常累积
- 用户进度可正常保存

### 15.2 演示验收

- 可演示完整流程：登录 -> 选关 -> 编码 -> 运行 -> 提交 -> 通关 -> 升级
- 页面风格统一，具备游戏化视觉特征
- 核心功能无明显阻塞性错误

## 16. 风险与限制

### 16.1 当前风险

- 关卡内容设计工作量较大
- 自动判题逻辑需要平衡准确性与实现复杂度
- 若首版强行实现后端代码运行，开发风险过高

### 16.2 处理建议

- 首版聚焦前端关卡
- 判题采用基础规则校验
- 后端与数据库学习区先作为地图占位内容

## 17. 后续迭代方向

- AI 教练系统
- 排行榜与成就系统
- React/Vue 专属关卡
- Node/Spring Boot 后端关卡
- 数据库实战关卡
- Boss 战与综合项目挑战

## 18. 结论

`WebQuest` 的核心不是单纯做一个在线编辑器，而是构建一个“可探索、可闯关、可成长、可持续激励”的 Web 开发学习平台。

MVP 阶段应聚焦“前端闯关 + 在线运行 + XP 成长 + 地图解锁”四个核心能力，优先保证产品闭环，后续再逐步扩展为完整的全栈学习冒险世界。
