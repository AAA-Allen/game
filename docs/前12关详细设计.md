# WebQuest 前 12 关详细设计

## 1. 文档目标

本文档基于 `课程路线图.md`，详细定义 `WebQuest` 第一章和第二章共 `12 关` 的教学内容、任务目标、前置知识、提示方式和判题方向。

本文件的用途是让后续开发可以直接进入以下工作：

- 关卡数据建模
- mock 题库补全
- 判题规则实现
- 地图与章节解锁逻辑设计

---

## 2. 适用范围

本文件覆盖以下两个教学区域：

- 第一章 `新手村：HTML 基础`
- 第二章 `CSS 森林：CSS 基础样式`

共计 `12 关`

补充说明：

- 它满足当前 `spec` 中“至少 10 个前端学习关卡”的 MVP 要求
- 同时也作为后续扩展到 30 关完整课程路线的第一阶段内容

---

## 3. 统一设计规则

### 3.1 关卡字段建议

每一关至少具备以下信息：

- `id`
- `zoneId`
- `chapter`
- `sortOrder`
- `title`
- `subtitle`
- `story`
- `learningObjectives`
- `prerequisites`
- `knowledgePoints`
- `taskDescription`
- `passCriteria`
- `hintLevels`
- `starterCode`
- `validationRules`
- `rewardXp`

### 3.2 难度分级

- `easy`：新概念首次出现
- `medium`：需要组合使用多个基础点
- `hard`：章节收尾或 Boss 关

### 3.3 提示分层

每一关建议至少提供 3 层提示：

- `提示 1`：方向性提示，不直接给答案
- `提示 2`：指出关键标签或样式
- `提示 3`：给出接近答案的实现思路

### 3.4 通关判定原则

- 先采用规则校验为主
- 优先校验“是否掌握目标知识点”
- 不用一开始就要求代码写法完全唯一

## 4. 第一章：新手村（HTML 基础）

### 4.1 章节目标

用户完成本章后，应具备以下能力：

- 理解网页由 HTML 结构组成
- 会使用常见文本标签
- 会插入图片和链接
- 会组织列表内容
- 会编写基础表单
- 能完成一个完整的静态介绍页面

### 4.2 章节 Boss 能力目标

本章 Boss 关结束后，用户应该能独立完成一个具备以下内容的网页：

- 标题和段落
- 头像图片
- 个人简介
- 技能列表
- 联系方式或链接

---

## 5. 第二章：CSS 森林（CSS 基础）

### 5.1 章节目标

用户完成本章后，应具备以下能力：

- 知道 CSS 如何选中元素
- 会调整文字样式
- 会设置背景和边框
- 会使用 margin 和 padding
- 理解 display 基础差异
- 能把 HTML 页面美化为像样的卡片界面

### 5.2 章节 Boss 能力目标

本章 Boss 关结束后，用户应该能把第一章做的内容美化成一个具有基础产品感的资料卡片页面。

### 基础信息

### 教学目标

- 认识网页的基础结构
- 知道页面中内容最终会展示在 `body` 中
- 学会写出一个标题和一个段落

### 前置知识

- 无

### 核心知识点

- `<main>`
- `<h1>`
- `<p>`

### 剧情背景

你刚进入新手村，村口的公告板还是空的。村长让你把欢迎信息写上去，让所有来到 WebQuest 的冒险者都知道这里是旅程的起点。

- 存在 `<main>`
- 存在 `<h1>`
- 存在 `<p>`

```html

</main>
```

```css
body {
  font-family: sans-serif;
}
```

```javascript
console.log("WebQuest ready");
```

### 判题方向

- 结构规则：检查是否存在 `main`、`h1`、`p`

- 知道元素大小和间距对阅读体验的影响

### 前置知识

- 关卡 9

### 核心知识点

- `width`
- `padding`
- `margin`
- 盒模型基础认知
